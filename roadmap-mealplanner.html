Éléments différenciants

🌍
Base de recettes mondiale
50 000+ recettes scrappées et structurées depuis Marmiton, Allrecipes, NYT Cooking, Serious Eats, bases API (Spoonacular, Edamam), et sources ethniques (japonais, mexicain, indien, libanais…). Vecteurs d'embedding pour recherche sémantique. Aucun concurrent FR n'a cette profondeur.
Base vectorielle · Embedding LLM
🧠
IA mémorielle des goûts
Chaque retour utilisateur (note, skip, "on a adoré") entraîne le profil de goût famille. Après 4 semaines, l'IA connaît mieux les goûts que le couple lui-même. Profils individuels par membre du foyer. Modèle collaboratif filtrant.
Collaborative filtering · Fine-tuning
🛒
Drive FR natif
Intégration directe Leclerc Drive, Auchan Drive, Intermarché Drive, Carrefour. Le panier se remplit en 1 clic depuis l'app. Substitution intelligente si produit indisponible. C'est le feature qui crée la rétention long terme — supérieure à tout autre.
Affiliation + API partenaires
📚
Livre de recettes hebdomadaire
Chaque dimanche, génération automatique d'un PDF/print imprimable : les 5-7 recettes de la semaine, avec photos générées, liste de courses, infos nutritionnelles, niveau de difficulté. Objet "tangible" à forte valeur perçue. Aucun concurrent ne fait ça.
PDF generation · Image AI
🥦
Mode frigo & anti-gaspi
Scan des produits du frigo (photo ou saisie). L'IA propose des recettes qui utilisent l'existant en priorité. "Restes du lundi" → recette du mardi. Réduction prouvée du gaspi alimentaire de 20-30%. Argument fort pour l'acquisition.
OCR · Vision AI
👨‍👩‍👧
Profils multi-membres
Chaque membre du foyer a ses propres restrictions, goûts et modes (enfants capricieux, sportif, végétarien, allergie). La planification réconcilie les contraintes automatiquement. Le partage en temps réel de la liste de courses évite les doublons et les "j'ai déjà acheté ça".
Multi-profil · Shared state
03

Stratégie base de recettes

Spoonacular API
~380 000 recettes
API complète avec données nutritionnelles, tags, temps de préparation, substitutions. Payant mais structuré. Base de départ solide.
Edamam API
~2M recettes
Forte couverture internationale, analyse nutritionnelle précise. Bon complémentaire pour les cuisines mondiales.
Marmiton Scraping
~70 000 recettes FR
Recettes françaises authentiques, notes communautaires. Scraping légal via robots.txt. Base essentielle pour la pertinence culturelle FR.
Open Food Facts
~3M produits
Base de données produits FR (EAN, Nutriscore, composition). Essentielle pour faire le lien recette → produit drive → panier.
Web Scraping ciblé
~30 000 recettes
NYT Cooking, Serious Eats, 750g, Ricardo, Allrecipes. Recettes de qualité éditoriale pour enrichir la bibliothèque premium.
LLM Generation
Illimité
Génération on-demand de recettes adaptées aux profils utilisateurs. Claude/GPT-4 pour créer des variations, adapter à des contraintes spécifiques, créer du contenu inédit.

📐 Pipeline de traitement des recettes

  • Normalisation des données : ingrédients → format canonique (nom, quantité, unité, catégorie)
  • Embedding vectoriel (OpenAI text-embedding-3 ou sentence-transformers) pour recherche sémantique
  • Tagging automatique : temps, difficulté, cuisine, régime, saison, budget estimé
  • Mapping ingrédient → produit Open Food Facts → SKU drive enseigne
  • Détection des doublons inter-sources (similarité cosine > 0.92)
  • Base vectorielle : Pinecone ou pgvector (PostgreSQL) pour les queries de similarité

05

Architecture agents IA

🔍
Agent · RECIPE_SCOUT
Collecte & enrichissement de recettes
Scrape les sources ciblées (Marmiton, 750g, web international), normalise les données, détecte les doublons via embedding, enrichit avec tags automatiques, valide la cohérence ingrédients/instructions. Tourne en batch nocturne.
🧬
Agent · TASTE_PROFILE
Moteur de recommandation personnalisé
Analyse les feedbacks (notes, skips, favoris), met à jour les vecteurs de préférence par profil, détecte les patterns familiaux, gère les contraintes contradictoires (enfant capricieux + parent veggie). Modèle collaboratif + content-based hybrid.
📅
Agent · WEEKLY_PLANNER
Planificateur hebdomadaire intelligent
Génère le plan de la semaine en tenant compte : profil goûts, régimes, temps de cuisine dispo, budget, saison, anti-répétition, réutilisation des restes, équilibre nutritionnel. Output : JSON plan + liste de courses consolidée.
🛒
Agent · CART_BUILDER
Constructeur de panier drive
Mappe chaque ingrédient vers le meilleur produit disponible dans l'enseigne choisie (via Open Food Facts + catalogue enseigne). Gère les substitutions si rupture, optimise les quantités (éviter le gaspillage), envoie le panier via API partenaire.
📚
Agent · BOOK_GENERATOR
Générateur du livre PDF hebdomadaire
Compile les 5-7 recettes de la semaine en PDF imprimable avec mise en page soignée, images (générées via Stability AI ou DALL-E), infos nutritionnelles, liste de courses et conseils de préparation. Génération automatique chaque dimanche soir.
🔔
Agent · RETENTION_LOOP
Anti-churn & engagement
Monitore les signaux de décrochage (inactivité 5j, baisse du taux d'ouverture), déclenche des actions proactives (notification push "nouveau plat selon tes goûts", email "ta semaine est prête"), gère les relances win-back. Règles + ML.

09

Challenges critiques

01

Le churn après 4–6 semaines

Les études montrent que les meal apps perdent 15–30% de leurs utilisateurs en mois 2 si les suggestions se répètent ou ne correspondent pas aux goûts réels. Solution : IA mémorielle active dès la semaine 1, relance proactive J+5 si inactivité, option "surprise me" pour briser la routine.

CRITIQUE
02

L'intégration drive : sans API publique officielle

Leclerc, Auchan et Intermarché n'ont pas d'API partenaire accessible directement. Jow a mis 2 ans à négocier ses partenariats. Stratégie : commencer par le scraping de panier (fragile mais fonctionnel) + démarcher commercialement en parallèle dès les premiers utilisateurs pour crédibiliser la proposition.

CRITIQUE
03

La qualité et la diversité des recettes

50 000 recettes de mauvaise qualité valent moins que 5 000 excellentes. Il faut un pipeline de validation rigoureux (LLM + humain pour les premières centaines), des photos attractives (génération IA ou partenariats), et une véritable diversité culinaire (pas que de la cuisine franco-française).

IMPORTANT
04

L'onboarding : le moment le plus fragile

Trop court → mauvaises recommandations → churn J+7. Trop long → abandon avant d'avoir vu la valeur. Benchmark Mealime : 3 questions, 60 secondes. PlateJoy : 50 questions, 5 minutes → trop long mais qualité supérieure. Trouver le bon équilibre, tester avec les 50 premiers beta users.

IMPORTANT
05

Le mapping ingrédient → produit → panier

"200g de lardons fumés" → quel produit exact chez Leclerc ? Quantité ajustée au format vendu ? Substitut si rupture ? Ce problème est sous-estimé mais c'est l'une des raisons pour lesquelles peu d'acteurs l'ont résolu proprement. Open Food Facts + base de données produits locale + validation manuelle initiale.

IMPORTANT
06

Différenciation vs Jow sur le long terme

Jow a 9M d'utilisateurs, 33M€ levés, et les mêmes partenariats drive. Le risque est de construire exactement le même produit en payant. La réponse : se différencier sur la qualité de l'IA (mémoire, diversité mondiale), le contenu premium (PDF, livre), et les profils famille avancés. Viser le segment "willing to pay" que Jow laisse intact.

IMPORTANT
